3 API Operations

All REST operations and general settings are described here.

3.1 General Options

Server Port

If your AR System Server requires a dedicated server port instead of the portmapper (0) the optional parameter port can be used to address the specific AR Server Port.

e.g. 

http://tomcat/arapi/arserver/HPD:Help%20Desk/'1'="INC000000000604"?port=12345

Available since version 0.4

Authentication

Authentication is done by HTTP Basic Authentication. User Name and Password entered in the application are passed to the AR System Server. Please make sure to use HTTPS / SSL encryption. Otherwise your Remedy Username/Password will be sent over the network in plain text.

The authentication realm is "myRealm".

Field Definition Cache (FieldIDs / FieldNames)

The field and schema information like field names & field ids are cached for better performance. The field cache is valid for 10 minutes and will be automatically refreshed after this time. The Field cache is server and user specific. This is important if AR RESTful API is used by different users with different permissions or on different servers with different permissions for the same user.

3.2 Query Data

Optional Parameters / General handling

The following optional parameters can be used to change the behavior of the GET operation.

Field IDs

By default the application returns field names. If required the parameter fieldNames=false can be provided in URL to return field ids instead. e.g. 

http://localhost:8080/arapi/arserver123.mydomain.de/HPD:Help%20Desk/'1'="INC000000000604"?fieldNames=false

For all POST / PUT operations you can address fields by name or id.

Selection Fields

By default the application returns the character presentation for all Selection Fields. E.g. The value Deployed for the field AssetLifecylcleStatus will be returned instead of its internal numerical value 3 . With the optional parameter translateSelectionFields=false this can be changed to return 3 instead.

e.g. 

http://localhost:8080/arapi/arserver123.mydomain.de/HPD:Help%20Desk/'1'="INC000000000604"?translateSelectionFields=false

Chunking

Chunking allows you to receive a large amount of records in small chunks. e.g. you want to receive all records in chunks with 1000 records each. The first query will set the parameter firstEntry to 0 and the parameter maxEntries to 1000. This will return records 1 - 1000. The second query will set the parameter firstEntry to 1000 and the parameter maxEntries to 1000. This will return the records 1001 - 2000.

Two optional parameters are available:

firstEntry: The first entry to return. Defaults to 0 maxEntries: The number of entries to return with this chunk. Defaults to 0 (all)

Samples

http://localhost:8080/arapi/arserver123.mydomain.de/HPD:Help%20Desk/'1' LIKE "INC00000000060%"?firstEntry=1000&maxEntries=1000

http://localhost:8080/arapi/arserver123.mydomain.de/HPD:Help%20Desk/'1' LIKE "INC00000000060%"?firstEntry=140

http://localhost:8080/arapi/arserver123.mydomain.de/HPD:Help%20Desk/'1' LIKE "INC00000000060%"?maxEntries=10

Attachment Fields

If an attachment is stored in the current record it will not be returned with the query operation. Instead a reference to the attachment will be provided. With a get operation on this reference the attachment can be received.

e.g.

Sample Query Response:

"Attachment Field":{
	"Name":"20140127_Existing_Records.xlsx",
	"URL":"http://localhost:8080/arapi/muc-vip-srvw084.intern.manyos.de/ARAPI:test/getAttachment/000000000000101/536870915"
}

The attachment can be received via a GET operation at the following URL:

http://localhost:8080/arapi/muc-vip-srvw084.intern.manyos.de/ARAPI:test/getAttachment/000000000000101/536870915

Count results only

A optional parameter countOnly is available to only return the number of records found by a query. Possible values are true, false. Default value is false.

Sample Response: 

{
  "form": "BMC.CORE:BMC_BaseElement",
  "query": "'DatasetId' = "BMC.ASSET" AND 'ClassId' != "BMC_BUSINESSSERVICE" AND 'AssetLifecycleStatus' = "Deployed"AND'ClassId'!="BMC_PERSON"",
  "resultSize": 20602
}

3.3 Create Data

Data can be create via a POST Request

Use the URL as follows

http://<tomcat>/arapi/<arserver>/<schema>

http://localhost:8080/arapi/arserver123.mydomain.de/HPD:Help%20Desk/

The data itself needs to be provided as JSON in the following format

{
	"<myReference>": {
		"<fieldid>": "<value>",
		"<fieldname>": "<value>"
	}
}

The response will return a JSON with the original reference and the created ar system request id

{
	"<myReference>": "<arentryid>"
}

e.g.

Sample Request

{
    "myID_00000000102": {
		"Short Description": "New Update Record",
		"Character Field": "rohase",
		"Decimal Field": 1.97,
		"Real Field": 63332.556,
		"Time Field": "19:00:00",
		"Integer Field": 9,
		"Diary Field": "New Entry",
		7: "Resolved",
		"Date Field": "01.06.2014",
		"Submitter": "roha",
		"Date/time Fiel": "Sun Jun 02 01:56:02 CEST 2014",
		"Currency Field": {
			"conversionDate": "Tue Apr 01 11:10:35 CEST 2014",
			"currencyCode": "USD",
			"value": 5299.82
		}
    },
    "myRequestNumber14": {
		"Status": 0,
		"8": "Shorty"
	}
}

Sample Response

{
	"myID_00000000102": "000000000000028",
	"myRequestNumber14": "000000000000031"
}

Attachments

Attachments can be set or update via a multipart/form-data POST request to a specific URL.

URL Format:

http://<tomcat>/arapi/<arserver>/<schema>/setAttachment/<entryID>/<attachmentFieldID>
  • <entryID> : The ID of entry / record where the attachment needs to be set / updated
  • <attachmentFieldID> : The Field ID of attachment field that needs to be set / updated

The filename needs to be provided as value for the name attribute.

Sample URL:

http://localhost:8080/arapi/muc-vip-srvw084.intern.manyos.de/ARAPI:test/setAttachment/000000000000101/536870915

3.4 Update Data

Data can be updated via a PUT Request

Use the URL as follows

http://<tomcat>/arapi/<arserver>/<schema>

http://localhost:8080/arapi/arserver123.mydomain.de/HPD:Help%20Desk/

The data itself needs to be provided as JSON in the following format

{
	"<requestid>": {
		"<fieldid>": "<value>",
		"<fieldname>": "<value>"
	}
}

The response will be in JSON and will return "success" or an error for each request

{
	"<requestid>": "success"
}

e.g.

Sample Request

{
	"000000000000028": {
		"Short Description": "New Update Record",
		"Character Field": "rohase",
		"Decimal Field": 1.97,
		"Real Field": 63332.556,
		"Time Field": "19:00:00",
		"Integer Field": 9,
		"Diary Field": "New Entry",
		7: "Resolved",
		"Date Field": "01.06.2014",
		"Submitter": "roha",
		"Date/time Fiel": "Sun Jun 02 01:56:02 CEST 2014",
		"Currency Field": {
			"conversionDate": "Tue Apr 01 11:10:35 CEST 2014",
			"currencyCode": "USD",
			"value": 5299.82
		}
    }, "000000000000028": {
		"7": "New",
		"8": "Moin"
	}

	"000000000000028": "success",
	"000000000000031": "success"

3.5 Delete Data

The DELETE operation deletes records based on a provided query.

Please keep in mind: The DELETE operation is not used much in Remedy / ITSM. Instead records are often only flagged as deleted (most often for audit / compliance). Also the delete operation might only work if your user has Administrator privileges.

Request Format:

http://<tomcat>/arapi/<arserver>/<schema>/<query>

e.g.

http://tomcat/arapi/arserver/ARAPI:test/'1' > "00000000000060%"

Sample Response

{
    "000000000000601": "success",
    "000000000000602": "success",
    "000000000000603": "success"
}

Optional Chunking

Chunking allows you to delete a large amount of records in small chunks. e.g. you want to delete all records in chunks with 1000 records each. The first query will set the parameter firstEntry to 0 and the parameter maxEntries to 1000. This will delete records 1 - 1000. The second query will set the parameter firstEntry to 1000 and the parameter maxEntries to 1000. This will delete the records 1001 - 2000.

Two optional parameters are available:

firstEntry: The first entry to delete. Defaults to 0 maxEntries: The number of entries to delete with this chunk. Defaults to 0 (all)

Samples

http://localhost:8080/arapi/arserver123.mydomain.de/ARAPI:test/'1' LIKE "00000000000060%"?firstEntry=1000&maxEntries=1000

http://localhost:8080/arapi/arserver123.mydomain.de/ARAPI:test/'1' LIKE "00000000000060%"?firstEntry=140

http://localhost:8080/arapi/arserver123.mydomain.de/ARAPI:test/'1' LIKE "00000000000060%"?maxEntries=10
